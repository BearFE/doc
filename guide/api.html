# API

# 联调配置

开发环境支持本地数据mock，远程代理，以及远程发布

1、mock 文件支持 js 和 json两种格式（具体格式参考 mock 下的文件），json 格式为静态数据，js 格式可模拟后端交互。

2、远程代理，异步数据支持远程代理，具体配置如下

先配置'config/proxy.js', 数据请求需要转发的server端机器地址，支持域名、ip、机器名访问

module.exports = {
    hosts: {
        test: {
            // 远程代理（数据转发），只要配置proxy字段即可，数据请求需要转发的server端机器地址，支持域名、ip、机器名访问
            proxy: 'http://bend.com:8500',
            receiver: 'http://bend.com:8500/receiver.php',
            path: '/home/users/bend/data'
        }
    },
    use: 'test'
};

然后配置mock下需要转发接口的'fullback'值为'true'，如获取插件list api

module.exports = function () {
    return {
        code: 0,
        msg: '',
        // 如需代理到远程机器，只要配置为true即可，如只要访问本地mock数据则配置为false
        fullback: true,
        data: {
            total: 20,
            pn: 1,
            ps: 14,
            list: [{
                pluginId: 4,
                name: 'bEnd插件',
                desc: 'bEnd解决方案，让B端平台用户降低理解成本，简化操作步骤，缩短跳转流程，从而快捷、方便、个性、定制化地享受线上运营服务',
                avatar: 'http://www.ld12.com/upimg358/allimg/c150617/143452O0515030-31O50.jpg',
                isAdd: 0,
                relApplies: [{
                    id: '1',
                    desc: '完成bEnd认证'
                },
                {
                    id: '2',
                    desc: '注册bEnd高级认证用户'
                }]
            }]
        }
    };
};

3、发布到远程机器，配置如下，支持watch

server端配置 https://github.com/fex-team/receiver 或 receiver.php，'receiver.php'脚本文件可以在项目根目录中找到；

前端配置'config/proxy.js'，如下：

module.exports = {
    hosts: {
        test: {
            proxy: 'http://bend.com:8500',
            // 发布依赖脚本，部署到远程机器上，可访问到即可，部署路径不限制，比如部署到odp环境static下面。
            receiver: 'http://bend.com:8500/receiver.php',
            // 静态资源及模版发布路径，与线上一致
            path: '/home/users/bend/data'
        }
    },
    use: 'test'
};

配置完成后，执行命令即可从本地动态发布到远程机器上，每次本地变更存储文件后动态rebuild，即watch

npm run deploy test

# 同步数据API

接口地址： /home ，访问的是'mock/home.json'文件，后续如是，不在赘述

接口参数： 同步数据，无

返回示例：

{
    
    // 状态码
    "code": 0,
    // 状态返回信息
    "msg": "success",
    "data": {
        // 用户信息
        "userInfo": {
            // 用户名称
            "name": "userName",
            // 头像
            "avatar": "http://www.ld12.com/upimg358/allimg/c150617/143452O0515030-31O50.jpg"
        },
        // 菜单信息
        "menu": {
            "pluginList": [{
                // 插件唯一标识
                "routerName": "example",
                // 插件中文名称
                "name": "插件",
                // 插件头图
                "avatar": ""
            }]
        }
    }
}

# 插件list

页面预览： /pluginlist

接口地址： /plugin/list

接口参数：

参数 类型 是否必须 描述
ps int 是 每页显示多少条
pn int 是 当前第几页

返回示例：

module.exports = function () {
    return {
        // 状态码
        code: 0,
        // 状态信息
        msg: '',
        // 如需代理到远程机器，只要配置为true即可，如只要访问本地mock数据则配置为false
        fullback: true,
        data: {
            // 总共多少条
            total: 20,
            // 当前第几页
            pn: 1,
            // 每页显示多少条
            ps: 14,
            // 菜单list
            list: [{
                // 插件id
                pluginId: 4,
                // 插件中文名称
                name: 'bEnd插件',
                // 插件描述
                desc: 'bEnd解决方案，让B端平台用户降低理解成本，简化操作步骤，缩短跳转流程，从而快捷、方便、个性、定制化地享受线上运营服务',
                // 插件头图
                avatar: 'http://www.ld12.com/upimg358/allimg/c150617/143452O0515030-31O50.jpg',
                // 插件是否已经添加，0:未条件，1:已添加
                isAdd: 0,
                // 插件添加条件
                relApplies: [{
                    id: '1',
                    desc: '完成bEnd认证'
                },
                {
                    id: '2',
                    desc: '注册bEnd高级认证用户'
                }]
            }]
        }
    };
};
