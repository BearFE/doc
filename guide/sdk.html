# SDK

使用场景，需要接入第三方插件开发的业务，供组件插件方式，第三方可按照规范编写满足自己需求的功能插件，这就依赖引入sdk作为中间的桥梁。

# SDK 源码

SDK提供两个功能，proxy（代理）、stat（统计），业务可以根据需要，按需扩展，Fork地址 GitHub

# proxy

api 异步代理，proxy 提供两个方法，get 和 post

import sdk from 'bend-sdk';

// * GET请求
sdk.proxy.get(
    // api
    'http://bend.com/api/test',
    {
        // get 只支持 params 参数模式
        params: {
            id: 123
        }
    }
).then(res => {
    if (res.status === 0) {
        // 正常返回 请求返回如果是json，返回的res.data为json格式
        console.log(res.data);
    }
}).catch(err => {
    // status 不为 0
    // code...
})

// * POST请求
sdk.proxy.post(
    // api
    'http://bend.com/api/test',
    {
        // data 将作为请求 body 发送
        // 常作为FormData, File, Blob发送
        data: {
            id: 123
        },
        // 或者也可以使用 params 传输，将作为 querystrng 的模式
        params: {
            id: 123
        }
        // 注意：data 和 params 选用一种即可
    }
).then(res => {
    if (res.status === 0) {
        // 正常返回
        console.log(res.data);
    }
}).catch(err => {
    // code ...
})

# stat

数据统计，第三方开发者开发插件中可以使用sdk提供的打点方式进行埋点，便于数据统计和效果的评估，提供主动调用addStat和点击事件代理两种方式

# 方法一：主动调用

用户在需要统计的地方，主动调用addStat方法

参数说明：

参数 类型 是否必须 描述
params Object 是 统计项

params 规范，规范可以根据实际业务更改，这里仅供参考

名称 类型 是否必须 描述
page string 是 页面名称，默认是插件名称 比如'home'首页
source string 是 页面来源
action string 是 触发时间，比如"click"
ext ... ... ...

示例：

import sdk from 'bend-sdk';

sdk.addStat({
    page: 'home',
    action: 'click',
    source: 'bend',
    ext: {
        isFirst: true
    }
});

# 方法二：页面代理

需要打点的标签上带上data-log属性，在点击的时候会自动代理发送请求

示例：

<div @click="" data-log="{page: 'home', ext: {isFirst: true}}"></div>
