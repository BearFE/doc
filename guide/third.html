# 第三方接入方案

第三方接入开发，涉及插件开发平台（B端）、插件市场到用户使用（C端）两部分，整体架构如下

开发者平台：分为用户中心（开发者用户管理）、开发文档（根据业务情况扩展本指南或者直接Fork使用）、产品管理（插件管理，例如一个产品线下面可以管理多个插件）

开发者平台-产品管理：每个插件都对应一套开发流程，包括开发、测试、发布、持续集成、入库等

# 代码托管方案&CR机制

1、直接使用 github管理，方便版本管理和代码CR

2、使用公司内部代码管理平台

3、发布zip包的形式，这种方式不利于代码CR

# 自动化沙盒

1、提供整个 bEnd脚手架给第三方开发使用

2、第三方插件统一在'src/plugins/xxx'目录开发插件，一个插件唯一入口文件，参照目录

3、沙盒机器上部署bEnd环境，第三方在开发者平台提交发布插件后，调用沙盒环境脚本，通过脚本部署第三方插件到沙盒机器上

# 持续集成方案

沙盒机器上审核及测试完成后，关联git产品库，直接push到产库即可

# 统计方案

参照SDK stat

# 运维

运维主要涉及到插件版本、降级方案、回滚方案、插件上线

# 降级方案

第三方插件上线后出现重大问题，影响面广，可以在插件平台增加人工干预，干预是否展现插件入口。整个bEnd架构，采用模块化开发模式，按需调用依赖插件静态资源，干预某个插件并不会影响整体环境。

# 回滚方案

第三方开发插件，增加版本管理，上线版本有问题，可以回滚到上一个版本

# 插件下线

1、开发者通过插件开发平台，请求插件下线

1、管理员关闭插件入口，可以通过权限模块干预入口下线

2、插件库代码模块删除
